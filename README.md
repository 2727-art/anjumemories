# ラスメモヴァンサバゲーム

Phaser を CDN で読み込む、ビルド不要の最小版 2D サバイバルゲームです。

## 起動方法

1. このフォルダをそのまま静的ファイルとして開ける状態にします。
2. `index.html` をブラウザで開きます。

補足:

- Phaser と画像読み込みの都合で、`file://` 直開きよりローカルサーバー経由の方が安定します。
- たとえば PowerShell なら、このフォルダで `python -m http.server 8000` を実行して `http://localhost:8000/` を開く方法が簡単です。

## 現在の最小実装

- `./画像/maincharactor/` の画像を使ったプレイヤー表示
- `./画像/skill/スキル1/skill_01.png` から `skill_08_frame_05.png` を使う 8 段階の基本スキル

- `./画像/monster/golem/sprite_0.png` から `sprite_4.png` を重装型 `Tank` の重い歩行アニメとして読み込む
- `./画像/backgrounds/shibuya_stage1/` に、植物に侵食された渋谷交差点風ステージ 1 の仮背景アセットを配置
- `./画像/backgrounds/shibuya_stage1/mockups/stage1_visual_target_01.png` に、参照画像の品質基準をこのゲーム向けに落とし込んだ完成イメージモックアップを保存
- 敵は `Chaser / Dash / Tank / Ranged` の 4 種類に増えており、スライム、ゴーレム、遠距離型アイコンで識別できる
- 経過時間に応じたウェーブ制で、出現敵の種類、比率、同時出現数が段階的に大きく変化する
- 3 分ごとに、強化されたエリート敵が 1 体だけ追加出現する
- 背景は地面タイル、デカール、障害物に分離して描画し、障害物だけ個別の当たり判定を持つ
- Stage 1 は `playBounds` で行動可能範囲を絞り、外周を高い廃ビルファサード、暗い外側マスク、透明コリジョン壁で囲う
- 遠距離型は一定距離を保ちながら弾を撃つ
- 近い敵へ自動で雷撃を落とす基本攻撃
- 接触ダメージと短い無敵時間
- 経験値ドロップとレベルアップ
- ゲーム開始時に、3回ぶんの初期強化を選択してからスタートできる
- 敵撃破時に、低確率で `Bronze / Silver / Gold` のレア経験値アイテムが落ちる
- レア経験値アイテムには、仮の正方形アイコンと色付きの光の柱エフェクトを付けている
- 敵撃破時に、低確率で回復・磁石・サポートアタック発動アイテムが落ちる
- 回復アイテムは HP を固定量だけ回復し、磁石アイテムは広範囲の経験値系アイテムを吸い寄せ、サポートアイテムはカットイン付きのサポートキャラクター攻撃を発動する
- 3 択の強化候補で、基本スキル強化に加えて竜巻スキルの解放と強化も選択可能
- 基本スキルはプレイヤーの周囲を周回する球体で、段階ごとにダメージ、接触間隔、球数、半径が成長
- 基本スキルは Stage 8 で4個になり、プレイヤーから離れたり近づいたりしながら旋回する
- 基本攻撃の雷撃はスキル段階に応じて同時本数が増える
- 竜巻スキルは画面内に留まりながら近い敵を追尾し、1体のまま段階ごとに追尾性能が成長
- 竜巻スキルは Stage 7 で敵を吸い込むフィールド効果を獲得
- 竜巻スキルは Stage 8 で大型化し、吸い込み範囲と吸引力がさらに伸びる
- 竜巻は移動時に風切り線を引き、吸い込み中の敵には減速感のある風エフェクトが出る
- 雷兎突進スキルは近い敵方向へジャンプしながら一方向に突進し、段階ごとの5フレーム素材を再生する
- 雷兎突進スキルは段階ごとにサイズ、距離、威力、再発動間隔が成長し、Lv8 で巨大化と2連続の着地雷衝撃を獲得する
- プレイヤーに追従するロボットが常時同行し、周囲の敵へミサイルを自動発射する
- ロボットのミサイルLvは金色宝箱、回復フィールドLvは銀色宝箱で最大Lv10まで成長する
- ミサイルLv10では通常ミサイルに加えてランダム着弾の大爆発ミサイルを発射する
- ロボットは回復フィールドを展開し、回復量と回復間隔はフィールドLvと壺強化で成長する
- ボス撃破時にロボット用の金/銀宝箱、金/銀壺が低確率でドロップし、壺では発射速度、攻撃力、HP回復間隔、回復量から1つを選択強化できる
- Stage 6 以降は球体が拡大縮小しながら周回し、発光と透明感も揺れて雷っぽく見える
- 周回球が敵に触れた瞬間に、短い発光リングとスパークのヒット演出を表示
- 敵は被弾時に少しノックバックし、撃破時に縮小しながら弾ける
- 経験値オーブは吸収時にトレイルを引き、取得時にプレイヤー周囲へ小さな発光パルスを表示
- HUD はポートレート、HP/XP バー、スキル枠、中央タイマー、ウェーブ/撃破数、目標パネル、ミニマップ、特殊アイテム枠を持つ商業ゲーム画面寄りの構成
- HUD フレームは `./画像/ui/hud/hud_overlay_frame.png` の透過 PNG を読み込み、金属フレームと黒いガラスパネル調で表示
- スキル枠では現在所持しているスキル段階を確認できる
- ベスト生存時間、最大到達レベル、最大撃破数、最大エリート撃破数を `localStorage` に保存
- ゲームオーバー時にプレイヤー名を入力して、モンスターキル数のローカルランキングを `localStorage` に保存
- Firebase Firestore と Anonymous Auth が有効な環境では、同じ記録をオンラインランキングにも投稿・表示
- ゲーム開始前にショップ画面を表示し、所持コインで CD 選択と永続強化を行える
- CD ショップは最大 10 枠の表示に対応し、現在は `Anju`、`なんでやねんねん`、`反省会`、`未来を生きてる` の 4 曲を登録
- 武器強化、鎧強化、靴強化を `localStorage` に保存し、次回以降のプレイ開始時ステータスへ反映
- 選択した CD の BGM をゲーム開始時にループ再生
- ゲームオーバー後の再スタート

## スキル定義の構造

`skillDefinitions.js` はビルド不要で読み込める外部データとして、`window.skillDefinitions` にスキル定義を置いています。

- ルートは `skillId` をキーにしたオブジェクトです
- `startsUnlocked` を `true` にすると、ゲーム開始時にそのスキルを所持します
- `behavior` でスキル挙動を切り替えます。現在は `orbit` と `screenHoming` を使っています
- `stages` は 1 段階ごとの配列で、`textureKey` と `imagePath` で見た目画像を指定します
- `displayScale` は見た目サイズ、`hitRadius` は当たり判定半径で、両方を分離して管理します
- `orbitCount`、`orbitRadius`、`rotationSpeed`、`spinSpeed`、`damage`、`contactTickMs`、`autoAttackShots` を段階ごとに変えることで、周回スキルと雷撃本数の成長を管理できます
- `unitCount`、`moveSpeed`、`idleRadius`、`screenPadding`、`trailIntervalMs` を使うと、画面内を追尾する竜巻系スキルを定義できます
- `animationFrames` を指定すると、スキル画像を複数フレームで再生できます
- `behavior: "directionalDash"` を使うと、`dashDistance`、`dashDurationMs`、`cooldownMs`、`jumpHeight`、`impactRadius` で一方向突進系スキルを定義できます
- `suctionRadius` と `suctionForce` を設定すると、竜巻の周囲に敵を引き寄せる効果を追加できます
- `orbitRadiusPulseAmount` を設定すると、周回スキルがプレイヤーから近づいたり離れたりする半径変化を追加できます
- `pulseScaleMin`、`pulseScaleMax`、`pulseSpeed` を設定すると、球体の見た目だけを脈動させられます
- 基本スキルでは Stage 1 の単球周回から始まり、Stage 8 で4球と半径変化を伴うMAX形態になります
- 竜巻スキルでは Stage 1 の単体追尾から始まり、Stage 7 で吸い込み、Stage 8 で大型吸引MAXになります
- 雷兎突進スキルでは Lv1 の読みやすい低速ジャンプ突進から始まり、Lv8 で巨大化と2連着地衝撃を追加します

## 敵とウェーブの構造

敵・ウェーブ・レア経験値アイテム・特殊アイテムの定義は `game.js` 先頭の定数で管理しています。

- `ENEMY_DEFINITIONS`: 敵 4 種類の基本性能と AI 用パラメータ
- `WAVE_DEFINITIONS`: 時間経過ごとの出現比率、スポーン間隔、同時出現上限
- `RARE_ITEM_DEFINITIONS`: Bronze / Silver / Gold の経験値量、ドロップ率、発光色
- `SPECIAL_ITEM_DEFINITIONS`: Heal / Magnet / Support の効果量、ドロップ率、見た目色、取得効果
- `SUPPORT_ATTACK_DEFINITIONS`: サポートキャラクターごとのカットイン、アニメ、効果範囲、持続時間、ダメージや状態異常を管理

## ステージ背景の構造

ステージ背景は `stageDefinitions.js` の `shibuyaStage1` 定義から読み込みます。

- `assets.ground`: シームレスに敷き詰める地面タイル
- `assets.decals`: 当たり判定を持たない散布用デカール
- `assets.obstacleTypes`: 障害物の見た目アセット定義
- `STAGE1_OBSTACLE_COLLISION_PRESETS`: 障害物ごとの当たり判定プリセット
- `STAGE1_OBSTACLE_LAYOUT_GROUPS`: 北側、南側、東側、西側、交差点角などの配置グループ
- `decalScatter`: seed 付きの半ランダム散布ルール
- `decalPlacements`: `id / type / sprite / x / y / scale / rotation / alpha / layer / zone` を持つ固定配置配列
- `obstaclePlacements`: `id / type / sprite / x / y / width / height / anchor / collision / layer / zone` を持つ固定配置配列
- `layout.crosswalks`: 横断歩道タイルを局所配置する帯定義
- `playBounds`: 行動可能範囲。敵スポーン、ミニマップ、透明な外周コリジョンの基準
- `boundaryPlacements`: 外周を囲う高い廃ビルファサード画像の配置

`game.js` 側は `decalPlacements` を優先して描画し、障害物だけ当たり判定を持ちます。`STAGE_DEBUG_OPTIONS` を有効にすると、レイヤー表示切替、障害物 hitbox、障害物ラベル、デカール数 / 障害物数の簡易表示で配置確認ができます。

現在の Stage 1 は「植物に侵食された渋谷交差点」モチーフで、中央は比較的開けた戦闘空間、外周寄りに障害物が増える構成です。

現在の敵仕様:

- `Chaser`: 標準速度と標準 HP の通常追尾型。スライムのぷるぷる待機アニメをそのまま使用
- `Dash`: HP は低いが、短い間隔で突進する高速型。スライムの待機アニメを暖色にして使用
- `Tank`: 低速高耐久で、ノックバック耐性が高い重装型。ゴーレムの低速歩行アニメを通常敵より一回り大きくして使用
- `Ranged`: 一定距離を保ちながら弾を撃つ遠距離型

現在のウェーブ仕様:

- 0:00-1:59 は `Chaser` 中心
- 2:00-3:59 で `Dash` を追加
- 4:00-5:59 で `Tank` を追加
- 6:00-7:59 で `Ranged` を追加
- 8:00 以降は混成ラッシュで密度も少し上がる
- 同時出現上限は `66 / 84 / 99 / 117 / 135` で、以前より敵数を多めにしている

エリートとレアアイテム:

- エリート敵は 3 分ごとに 1 体だけ出現し、HP、サイズ、経験値量、オーラを強化してあります
- レア経験値アイテムは敵撃破時の低確率ドロップで、`Gold > Silver > Bronze` の順に経験値量が高いです
- 通常敵のレアドロップ率は `Bronze 0.6% / Silver 0.22% / Gold 0.07%` に絞っています
- エリート敵撃破時のみ、追加で `Bronze +1.2% / Silver +0.5% / Gold +0.2%` の補正が入ります
- レアアイテムは仮の正方形アイコンと、色付きの縦長ビームで見つけやすくしています

特殊アイテムと記録保存:

- `Heal`: 通常敵からは低確率、エリートからは少し高めでドロップし、取得時に HP を固定量だけ回復します
- `Magnet`: 通常敵からは低確率、エリートからは少し高めでドロップし、広範囲の経験値オーブと金銀銅レア経験値アイテムを吸い寄せます
- `Support`: 通常敵からはかなり低確率、エリートからは少し高めでドロップし、ランダムなサポートキャラクターのスキルを発動します
- 特殊アイテムは 1 体の敵から最大 1 個までで、マップ上の同時存在数も増えすぎないよう軽く制御しています
- ベスト記録は `localStorage` の `lastmemoVansabaBestRecord` に JSON で保存し、`survivalTimeMs / level / kills / eliteKills` を保持します
- ゲームオーバー時に現在 run とベスト記録を比較し、更新があれば次回起動後も残ります

Firebase オンラインランキング:

- Firebase 側で Firestore Database を作成し、Authentication の匿名ログインを有効化してください
- GitHub Pages で公開する場合は、Firebase Authentication の承認済みドメインに GitHub Pages のドメインを追加してください
- Firestore Rules には、このフォルダの `firestore.rules` の内容を設定してください
- オンラインランキングは Firestore の `leaderboardKills` コレクションを使います
- Firebase への接続や書き込みに失敗した場合でも、ローカルランキングはそのまま保存・表示されます

今後スキルを追加する手順:

1. `./画像/skill/` 以下に画像を追加します
2. `skillDefinitions.js` に新しい `skillId` を追加し、各段階の `imagePath` と性能を定義します
3. 初期所持にしたい場合だけ `startsUnlocked: true` を付けます
4. 別入手にしたい場合は、`game.js` 側で解放条件や報酬処理を追加します
